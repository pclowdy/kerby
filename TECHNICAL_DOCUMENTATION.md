# K8s Microservices Application - Technical Documentation

## 1. Executive Summary
The K8s Microservices Application is a robust, scalable backend system paired with a premium React frontend. It functions as a secure e-commerce marketplace and administrative inventory management system. Key features include consumer and admin portals, real-time stock tracking, automated order processing, and live notifications. By leveraging a microservices architecture on Kubernetes, the platform ensures high availability, independent component scalability, and strict fault isolation, solving the problem of monolithic bottlenecks in traditional marketplace applications.

## 2. Tech Stack & Tools

**Backend Services:**
- **Go (Golang)**: Chosen for its fantastic performance, strong concurrency model, and seamless gRPC support.
- **gRPC & Protocol Buffers**: Used for fast, strongly-typed internal communication between microservices (Auth, Order, Inventory).
- **HTTP / REST / SSE**: Used at the API Gateway to interface with the frontend and proxy notifications.

**Frontend:**
- **React 18 & Vite**: For building a lightning-fast, modern Single Page Application (SPA).
- **Vanilla CSS**: Provides a dynamic, glassmorphic luxury design without heavy framework overhead.

**Data Storage & Messaging:**
- **PostgreSQL**: The primary relational database for structured, transactional data (Users, Orders, Inventory).
- **Redis**: Used as a Pub/Sub message broker to handle real-time event broadcasting (e.g., `ORDER_PLACED`, `LOW_STOCK`) for notifications.

**Infrastructure & Deployment:**
- **Docker**: Containerization of all Go services and frontend.
- **Kubernetes (K8s)**: Orchestration of containers ensuring self-healing, load balancing, and configuration management.
- **Nginx Ingress**: Routes external traffic into the K8s cluster and directs it to the appropriate services.

## 4. File-by-File Breakdown

### Root Scripts
- **`build_and_deploy.ps1`**: A PowerShell script automating the local deployment pipeline. It builds Docker images for all 5 services, loads them into a local `kind` cluster, and applies all K8s manifests.
- **`cloud_deployment_guide.md`**: Instructions for taking the app live on a cloud provider (e.g., GKE, DigitalOcean) including updating registry paths and ingress adjustments.
- **`generate.sh`**: A bash script that invokes `protoc` to generate Go code from the `.proto` files, maintaining the gRPC contracts.

### Frontend
- **`frontend/src/App.jsx`**: The core React application containing multiple components:
  - `Login`: Handles user authentication and secure bcrypt-backed registration. 
  - `MainLayout`: The application shell with navigation and the real-time notification drawer.
  - `Dashboard`: Consumer view to place orders.
  - `Orders`: Displays consumer's order history.
  - `Inventory`: Admin view to manage and replenish stock.
  - `Profile`: User metrics, Role-Based Access Control verification, and logout functionality.
- **`frontend/src/index.css`**: The design system implementing the glassmorphic, luxury aesthetic with custom animations and variables.
- **`frontend/vite.config.js`**: Build tooling and development server configuration, including routing proxies.

### K8s Manifests
- **`k8s/applications/microservices.yaml`**: Contains the Kubernetes Deployments and Services for API Gateway, Auth, Inventory, Order, and Notification services.
- **`k8s/config/configmap.yaml` & `secret.yaml`**: Configures environment variables (Service Hosts, Ports) and injects sensitive credentials (DB Passwords) securely into pods.
- **`k8s/databases/postgres.yaml` & `redis.yaml`**: Stateful component deployments and internal services for Postgres and Redis.
- **`k8s/networking/ingress.yaml`**: Networking rules mapping HTTP URL paths (like `/api/v1/*`) to the correct internal K8s services.

### Protobuf Contracts (`proto/`)
- **`auth.proto`, `inventory.proto`, `order.proto`**: Define the strict RPC methods (e.g., `CheckStock`, `CreateOrder`, `Login`) and data structures used for lightning-fast internal service-to-service communication.

### Go Microservices (`services/`)
- **`api-gateway/main.go`**: The entry point for the frontend. Translates HTTP/REST requests into gRPC calls to backend services, handles CORS, and extracts & validates JWTs to protect downstream endpoints.
- **`auth/main.go`**: Manages secure user registration and login functionalities. It hashes incoming passwords with `bcrypt`, stores them in PostgreSQL, blocks duplicate usernames, and generates JWT tokens for authenticated sessions.
- **`inventory/main.go`**: Centralized stock manager. Prevents overselling by running atomic updates queries (`UPDATE ... RETURNING`) on the Postgres database. Exposes vital gRPC methods for the core application like `CheckStock` and `DeductStock`.
- **`order/main.go`**: Orchestrates the purchasing workflow constraint. Upon receiving a request, it performs an initial stock check via gRPC, securely saves the new order in Postgres, performs the actual stock deduction via gRPC, and finally triggers an `ORDER_PLACED` event onto Redis Pub/Sub.
- **`notification/main.go`**: A lightweight notification daemon that constantly subscribes to the `orders` Redis channel. It caches recent real-time alerts (`ORDER_PLACED`, `LOW_STOCK`) and serves them to the frontend upon request.

## 5. Data Flow & Architecture

1. **User Request**: The user interacts with the React frontend, sending an HTTP REST request to the exposed Nginx Ingress Controller.
2. **Routing**: The Ingress controller maps the `/api/v1/*` endpoint to the `api-gateway` K8s service.
3. **Authentication**: For protected routes (like ordering or checking personal orders), the API Gateway extracts the JWT bearer token and sends a gRPC validation request off to the Auth Service.
4. **Processing (e.g., Making a Purchase)**:
   - The API Gateway executes a `CreateOrder` gRPC call to the Order Service.
   - The Order Service fires a `CheckStock` gRPC call to the Inventory Service.
   - Provided stock is available, the Order Service persists the row in PostgreSQL, logs the transaction, and executes `DeductStock`.
5. **Event Publishing**: Order Service publishes a message (e.g., `ORDER_PLACED` and potentially `LOW_STOCK` if the remaining quantity drops below 5) to the Redis Pub/Sub network.
6. **Notification**: The Notification Service actively listening on Redis grabs the event, saving it to localized state (or a Redis List) and serving it to the frontend's visual notification drawer on the next polling interval.

## 6. Setup & Deployment

> [!IMPORTANT]
> Ensure you have Docker running locally, and a local Kubernetes cluster tool like `kind`, `minikube`, or Docker Desktop enabled K8s installed. You will also need `kubectl` contextually aimed at your cluster.

**Step-by-Step Local Deployment:**

1. **Clone & Navigate**:
   Open a terminal and navigate to the root directory `k8s-microservices-app`.

2. **Execute Deployment Script**:
   Run the localized PowerShell script. This orchestrates the compilation of Docker images across all 5 Go microservices, seamlessly loads them into the `kind` cluster logic, and natively provisions the defined K8s infrastructure schemas.
   ```powershell
   ./build_and_deploy.ps1
   ```

3. **Verify Cluster Readiness**:
   Monitor the cluster pods until everything hits the `Running` state. Wait for the database pods to execute initialization commands cleanly before proceeding.
   ```bash
   kubectl get pods -w
   ```

4. **Access the Web Application**:
   If utilizing Docker Desktop (with default LoadBalancers routing), the app may directly map at `http://localhost`. Otherwise, manually port-forward the API Gateway for testing via:
   ```bash
   kubectl port-forward svc/api-gateway 8080:80
   ```
   Deploy the Vite Frontend server for isolated frontend development:
   ```bash
   cd frontend
   npm install
   npm run dev
   ```

5. **Utilize the System**:
   - Register a new account digitally from the index screen under the "Consumer" scope.
   - Log out to explore testing the "Admin" portal access by utilizing the unified `admin` default superuser. 
   - Interact via the unified Dashboard!
